# The Atomic Dial
### by Yonathan Gur, Ryan Gelston, Ty Bequette, and Cody Fuh

---

## Overview
The Atomic Dial is an open-source, haptic-enabled smart knob powered by the RP2350B architecture. It provides software-defined mechanical feedback—such as detents, virtual end-stops, and magnetic resistance—using a brushless DC motor. An adaptation of the [SmartKnob](https://github.com/scottbez1/smartknob) project by Scott Bezek, this project adapts the original device to work smoothly with the RP2350B microcontroller. The board was designed to include header pins to plug in the [Proton development board](https://ece362-purdue.github.io/proton-labs/) designed by staff for ECE 36200: Microprocessor Systems and Interfacing at Purdue University.

## Board Design & Hardware Integration
Transitioning from the original ESP32-PICO architecture to the RP2350B required a redesign of the Base PCB and hardware routing to accommodate the Proton development board. We designed a custom baseplate PCB that features female headers specifically spaced to receive the Proton board. These header pins allow anyone with a Proton board to easily recreate this project and use it with their own boards. A bulk of the work done here was carefully rerouting the signals for each of the peripherals to match the correct function needed for compatibility with the proton board (see [electronics/ProjectPinAssignments.xlsx](https://github.com/Ty-Bequette-217/atomic_dial/blob/main/electronics/ProjectPinAssignments.xlsx)).

## Firmware Architecture
To make the RP2350B drive a display, read sensors, and commutate a motor simultaneously without stuttering, the firmware is divided into distinct, non-blocking layers built on the Raspberry Pi Pico C/C++ SDK.

### AS5600 Magnetic Encoder
The AS5600 serves as the primary feedback loop for our haptic system. Mounted directly below the diametric magnet on the motor shaft, it continuously reads the absolute angular position of the knob. By feeding this high-resolution rotational data back into the RP2350B, the firmware can calculate exact phase angles and trigger localized spikes in motor torque, allowing the user to feel physical "clicks" entirely generated by software.

### GC9A01 Round LCD
The primary user interface is driven by a 1.28-inch GC9A01 circular LCD, which receives data over the RP2350B's hardware SPI bus to maintain a high framerate.

#### Initialization for GC9A01
Bringing up the GC9A01 requires a precise sequence of initialization commands sent over SPI. The firmware configures the display out of sleep mode, sets the pixel format to 16-bit color (RGB565), defines the memory access control for correct screen orientation, and establishes the circular column and row address windows. A hardware reset line is toggled on boot to ensure the display controller starts in a known, clean state before accepting pixel data.

#### Inclusion of LVGL C Library
To handle the complex graphics, menus, and animations without writing pixel-level drawing routines from scratch, we integrated the [LVGL (Light and Versatile Graphics Library)](https://lvgl.io/). LVGL operates using internal draw buffers. We implemented a custom `disp_flush` callback function that takes LVGL's rendered buffer blocks and pushes them over the SPI bus to the GC9A01. This allows us to create smooth, professional-looking UI dials and scrolling menus that react in real-time to the motor encoder.

### TMC6300 Motor Driver
The physical haptic feedback is generated by a gimbal brushless DC motor driven by the TMC6300. The RP2350B uses its hardware PWM slices to output precisely timed, 3-phase sinusoidal waveforms (Space Vector PWM) to the motor coils. Because the PWM generation is handled by hardware timers rather than the main CPU loop, the motor commutation remains completely jitter-free even while the processor is busy rendering graphics or reading sensors.

### HX711 Load Cell Amplifier
To add a "push-to-click" dimension to the dial, we integrated SMD resistors acting as load cells monitored by an HX711 24-bit ADC. When the HX711 outout decreases below the threshold, a push is registered and changes the state of the device.

### LDR monitoring for LCD Brightness
To optimize visibility and power consumption, the dial features dynamic backlight adjustment. A Light Dependent Resistor (LDR) acts as an ambient light sensor, read continuously by the RP2350B's internal 12-bit ADC. The firmware maps these raw ADC values to a duty cycle percentage, which is then fed into a hardware PWM slice driving the `LCD_BACKLIGHT` pin. This ensures the display dims in dark rooms and ramps to full brightness in heavily lit environments without any manual user intervention.
